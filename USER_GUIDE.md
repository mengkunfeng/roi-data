# ROI 数据分析系统 - 用户使用指南

## 1. 系统概述

ROI 数据分析系统是一个专门用于分析应用安装 ROI（投资回报率）数据的工具，帮助用户直观了解不同时间点的 ROI 变化趋势，为决策提供数据支持。

### 1.1 主要功能

- **数据自动导入**：每天凌晨 2 点自动导入 CSV 数据
- **数据可视化**：通过多种图表展示 ROI 数据
- **数据筛选**：支持按应用、国家地区等维度筛选数据

### 1.2 系统要求

- **前端**：现代浏览器（Chrome、Firefox、Safari、Edge）
- **后端**：Node.js 18+、MySQL 5.7+
- **网络**：稳定的网络连接

## 2. 快速开始

### 2.1 系统部署

#### 2.1.1 后端部署

1. 安装依赖

   ```bash
   cd roi-express
   npm install
   npm install node-cron
   ```

2. 配置数据库

   - 编辑 `src/config/db.ts` 文件，设置数据库连接信息
   - 确保 MySQL 数据库已创建 `roi_data` 表

3. 启动服务
   ```bash
   npm run build
   npm start
   ```

#### 2.1.2 前端部署

1. 安装依赖

   ```bash
   cd roi-data-analysis
   pnpm install
   ```

2. 启动开发服务器

   ```bash
   pnpm dev
   ```

3. 访问应用打开浏览器访问 http://localhost:8000

### 2.2 数据准备

1. 准备 CSV 数据文件
2. 将文件放入 `roi-express/src/assets/` 目录
3. 系统会在每天凌晨 2 点自动导入数据

## 3. 功能说明

### 3.1 数据导入

#### 3.1.1 自动导入

- **时间**：每天凌晨 2 点
- **流程**：扫描 CSV 文件 → 解析数据 → 导入数据库 → 删除 CSV 文件
- **文件格式**：
  ```
  日期,app,出价类型,国家地区,应用安装.总次数,当日ROI,1日ROI,3日ROI,7日ROI,14日ROI,30日ROI,60日ROI,90日ROI
  ```

#### 3.1.2 手动导入（可选）

1. 编辑 `roi-express/src/app.ts` 文件
2. 在启动时调用 `importCSVToDatabase()` 函数
3. 重启后端服务

### 3.2 数据查询

- **API 接口**：`GET /api/roi`
- **响应格式**：JSON 格式，包含所有 ROI 数据
- **数据更新**：每次导入后自动更新

### 3.3 数据可视化

#### 3.3.1 图表类型

- **折线图**：展示 ROI 随时间的变化趋势

#### 3.3.2 筛选功能

- **按应用筛选**：选择特定应用查看数据
- **按国家地区筛选**：选择特定国家地区查看数据

## 4. 使用方法

### 4.1 首次使用

1. 访问系统 http://localhost:8000
2. 上传 CSV 数据文件到后端 assets 目录
3. 等待系统自动导入数据（或手动触发导入）
4. 查看数据可视化结果

### 4.2 日常使用

#### 4.2.1 查看数据

1. 访问系统首页
2. 系统会自动加载最新的 ROI 数据
3. 使用筛选器选择查看特定数据
4. 查看不同类型的图表展示

#### 4.2.2 更新数据

1. 将新的 CSV 数据文件放入后端 assets 目录
2. 系统会在每天凌晨 2 点自动导入
3. 导入完成后，刷新前端页面查看最新数据

## 5. 常见问题

### 5.1 数据导入问题

#### Q: 数据没有自动导入？

**A:** 检查以下几点：

- 后端服务是否正在运行
- CSV 文件是否放在正确的目录（`roi-express/src/assets/`）
- CSV 文件格式是否正确
- 数据库连接是否正常
- 查看后端日志，检查是否有错误信息

#### Q: 导入后数据显示不正确？

**A:** 检查以下几点：

- CSV 文件格式是否符合要求
- 数据类型是否正确（尤其是日期和数字格式）
- 数据库表结构是否正确

### 5.2 前端显示问题

#### Q: 前端页面显示空白？

**A:** 检查以下几点：

- 前端服务是否正在运行
- 后端服务是否正在运行
- 浏览器控制台是否有错误信息
- API 接口是否正常响应

#### Q: 图表没有显示数据？

**A:** 检查以下几点：

- 后端是否已成功导入数据
- API 接口是否返回数据
- 浏览器控制台是否有错误信息
- 筛选条件是否正确

### 5.3 系统性能问题

#### Q: 页面加载缓慢？

**A:** 可能的原因：

- 数据量过大
- 网络连接不稳定
- 浏览器缓存问题
- 服务器性能问题

**解决方法：**

- 优化数据查询
- 增加服务器资源
- 清理浏览器缓存
- 考虑分页加载数据

## 6. 故障排除

### 6.1 后端服务故障

#### 6.1.1 服务无法启动

- **检查**：Node.js 版本是否符合要求
- **检查**：依赖包是否安装完整
- **检查**：数据库连接是否配置正确
- **检查**：端口是否被占用

#### 6.1.2 定时任务不执行

- **检查**：node-cron 依赖是否安装
- **检查**：定时任务配置是否正确
- **检查**：服务是否一直在运行（没有被重启）
- **检查**：日志文件查看错误信息

### 6.2 数据库故障

#### 6.2.1 连接失败

- **检查**：数据库服务是否正在运行
- **检查**：连接配置是否正确
- **检查**：数据库用户权限是否足够
- **检查**：网络连接是否正常

#### 6.2.2 数据导入失败

- **检查**：CSV 文件格式是否正确
- **检查**：数据库表结构是否正确
- **检查**：数据库空间是否足够
- **检查**：是否有主键冲突

### 6.3 前端故障

#### 6.3.1 页面无法访问

- **检查**：前端服务是否正在运行
- **检查**：端口是否正确
- **检查**：浏览器是否支持
- **检查**：网络连接是否正常

#### 6.3.2 功能无法使用

- **检查**：浏览器控制台是否有错误信息
- **检查**：API 接口是否正常响应
- **检查**：前端代码是否有语法错误
- **检查**：依赖包是否安装完整
